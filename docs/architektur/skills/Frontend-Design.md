# Frontend-Design

> ⬆ [[00 Skills Übersicht]] · [[../index]]

Produktionsreife, distinctive Frontend-Interfaces mit hoher Designqualität.

## Design-Philosophie

### Vor dem Code
- **Purpose**: Welches Problem löst das Interface? Wer nutzt es?
- **Tone**: Eine klare Richtung wählen: brutal minimalist, maximalistisch, retro-futuristisch, organisch/natürlich, luxury/refined, playful/toy-like, editorial/magazine, brutalist/raw, etc.
- **Constraints**: Technische Anforderungen (Framework, Performance, Accessibility)
- **Differentiation**: Was macht das UNVERGESSLICH?

### Code-Prinzipien
- Produktionsreife, funktionierende Implementierung
- Visuell ansprechend und einprägsam
- Kohärent mit klarer Design-Position
- Sorgfältig verfeinert in jedem Detail

## Frontend-Ästhetik-Guidelines

### Typografie
- Fonts wählen, die schön, einzigartig und interessant sind
- Generische Fonts (Arial, Inter) vermeiden
- Distinktive Display-Fonts mit raffinierten Body-Fonts paaren

### Farbe & Theme
- Kohäsives Ästhetik-Commitment
- CSS-Variablen für Konsistenz
- Dominante Farben mit scharfen Akzenten

### Motion
- CSS-only Lösungen priorisieren
- High-Impact Momente: gestaffelte Page-Load-Reveals
- Hover-States die überraschen

### Spatial Composition
- Unerwartete Layouts, Asymmetrie, Überlappung
- Grid-brechende Elemente
- Großzügiger Negative Space ODER kontrollierte Dichte

### Backgrounds & Visual Details
- Gradient Meshes, Noise Textures, geometrische Patterns
- Layered Transparencies, dramatische Schatten
- Kontextuelle Effekte die zur Ästhetik passen

## Was NICHT tun
- Generische AI-Ästhetik vermeiden
- Keine Inter, Roboto, Arial, System-Fonts
- Keine generischen Purple-Gradients
- Keine vorhersehbaren Layouts
- Nicht zu generischen Font-Wahl konvergieren (Space Grotesk vermeiden)
